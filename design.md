# 幕後 — 设计文档

> 最后更新：2026-06-23

---

## 一、产品理念

**幕後** 的核心理念：你不是在"过生活"，你是在"制片"你的人生。

就像一个电影或展览的幕后制作团队，这个工具帮你管理创作过程中的所有碎片——目标、灵感、习惯、日记、财务——让它们不再是孤立的信息，而是一个互相联动的系统。

### 设计原则

1. **All-in-one**：一个页面搞定所有事，不用在多个工具间切换
2. **数据联动**：OKR 完成任务 → 自动写日记；习惯打卡 → 复盘统计。数据自己会流动，不用重复填
3. **手动优先 + AI 辅助**：核心数据自己掌控，AI 负责分析和建议，不替你做决定
4. **低摩擦**：快速捕捉不需要分类、打标签，先记下来再说；复盘数据自动汇总，不用手动翻查
5. **轻量**：单文件、纯前端、免费数据库。不需要服务器，不需要运维

## 二、信息架构

### 导航（8 项，无二级菜单）

```

├── 🎯 OKR+习惯     ← 默认首页，目标管理 + 习惯打卡 + 今日3件事 + 备忘录
```

## 三、页面设计

### 1. OKR+习惯（首页）

**布局**：左右分栏（OKR 70% + 习惯 30%）

左栏内容：
- 座右铭（可点击编辑）
- 今日3件事 + 备忘录（并排）
- 周期选择器（季度 + 月份）
- OKR 四级列表（O→KR→任务→子任务），每项有进度条 + 勾选框

右栏内容：
- 习惯月历 + 年热力图
- 今日习惯打卡列表

**关键交互**：
- 任务打勾 → 自动写入日记时间线
- 今日3件事从 OKR 层级弹窗中选取
- OKR 目标采用两列网格（大屏）排列

### 2. 愿景版

**布局**：软木板风格自由画布

- 两个 Tab：N 年愿景 / 1 年愿景
- 便签可拖拽、自由定位
- 每个便签：标题 + 描述 + 图片链接
- 右下角 "+" 按钮新建便签

### 3. 灵感库

**布局**：卡片网格 + 顶部统计条

统计条内容：
- 总收集数 | 各状态数量（刚收集/已做笔记/已实践/已废弃）| 标签数量

卡片网格：
- 状态筛选按钮（全部/刚收集/已做笔记/已实践/已废弃）
- 模糊搜索框
- 每张卡片：标题、备注、标签、状态、时间
- 支持图片上传和展示

### 4. 日记

**布局**：月历 + 日记内容

- 左侧月历导航，有日记的日期显示标记
- 右侧当日日记：心情选择 + 时间线（自动记录 OKR 完成/习惯打卡）+ 自由书写区
- 支持图片上传

### 5. 复盘（月度）

**布局**：左栏（数据详情）+ 右栏（总览 + 财务 + 反思）

**右栏内容（从上到下）**：
1. 数据总览卡片：OKR完成率 / 习惯打卡天数 / 日记篇数 / 灵感实践 / 备忘完成
2. 💰 财务回顾：左数字（收入/支出/结余/本月存/累计存）+ 右文字（自由书写）
3. 4 步引导式反思：成就感 / 困难 / 学到什么 / 下月重点

**左栏内容**：
- 月份选择器
- 历史复盘列表
- 详细数据：各目标进度条 / 本月完成任务列表 / 习惯打卡明细 / 灵感动态

**关键设计决策**：
- 财务回顾放在数据卡片和反思问题之间，起到"从客观数据过渡到主观反思"的桥梁作用
- 结余自动计算（收入 - 支出），不需要手动算
- 累计存款是手动填的（不是跨月自动累计），每个月自己更新

### 6. 捕捉·回顾

**布局**：左右 1:1 网格

左 — 快速捕捉：
- 大输入框 + Ctrl+Enter 快捷提交
- 最近捕捉列表（可删除）

右 — 随机回顾：
- 类型筛选（全部/日记/灵感/成就）
- 塔罗式卡片（背面图案 + 翻牌动画 + 粒子特效）
- "抽一张"按钮
- 抽卡历史列表

**设计决策**：这两个功能合并在一个页面，因为它们都是"轻量操作"——不需要深度编辑，随来随走。

### 7. AI 助手

- 5 个快捷按钮：本周心情 / 灵感建议 / 跨界连接 / 本周小结 / OKR 诊断
- 自由对话输入框
- 聊天式展示，支持清空
- 后端接 DeepSeek API

### 8. 设置

- Supabase URL + 密钥（预填）
- 简洁表单，不需要多余说明

## 五、视觉设计

### 设计风格

**方向**：温暖 + 克制 + 一点仪式感

不是冷冰冰的 productivity tool，更像一本私人创作手账。配色偏暖，卡片有圆角，有微妙的阴影和过渡动画。

### 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色（accent） | `#E76F51` | 暖橙，用于重点按钮、进度条、选中态 |
| 背景（bg） | `#FAFAF7` | 暖白底 |
| 卡片（surface） | `#FFFFFF` | 纯白卡片 |
| 边框（border） | `#E8E4DF` | 浅暖灰边框 |
| 主文字（text） | `#3D3D3D` | 深灰，比纯黑柔和 |
| 辅助文字（muted） | `#999999` | 提示、说明 |

### 排版

| 层级 | 字号 | 字重 |
|------|------|------|
| Logo | 24px | 700 |
| 页面标题 | 24px | 700 |
| 区块标题（h3） | 14-15px | 600 |
| 正文 | 13-14px | 400 |
| 辅助文字 | 11-12px | 400 |

字体：系统默认（`-apple-system, ...`），中文用系统自带中文字体。

### 圆角与间距

- 大卡片：`border-radius: 12-16px`
- 小元素：`border-radius: 6-8px`
- 区块间距：`margin-bottom: 20-28px`
- 内边距：`padding: 16-28px`

## 六、技术架构

### 整体架构

```
浏览器（单 HTML 文件）
├── 侧边栏导航 → 页面切换（显示/隐藏 page div）
├── Supabase JS SDK → PostgreSQL 数据库（云端）
├── DeepSeek API → AI 分析（仅 AI 助手页面调用）
└── localStorage → 座右铭（本地持久化）
```

### 关键参数

- **Supabase 项目**：`otbzfmfqtfcdlronivdk`
- **数据库**：PostgreSQL（Supabase 免费额度）
- **部署**：GitHub Pages，legacy 模式（从 main 分支根目录部署）
- **线上地址**：`https://kahoyoyo143-eng.github.io/muhou/`

### 部署流程

```
修改 index.html → git add + commit → git push → GitHub Pages 自动部署 → 约 30 秒生效
```

不需要 GitHub Actions（legacy 模式不需要 workflow），不需要 build 步骤（纯静态 HTML）。

## 七、关键设计决策记录

### 为什么把标签管理页删了？
标签管理（改名/合并/删除）对于个人使用场景使用频率太低，而且增大了页面复杂度。标签的实用信息（有哪些标签、每个标签下多少条）用一个统计条放在灵感库顶部就够了。

### 为什么把快速捕捉和随机回顾合并？
两个功能都是"低摩擦操作"——不需要深度编辑，随来随走。合并在一个页面减少导航项，左捕右顾的布局也符合使用心理：先捕捉新东西，再回顾旧东西。

### 为什么财务回顾放在复盘里而不是独立页面？
财务复盘的意义在于和其他维度（OKR完成、习惯打卡、灵感实践）放在一起看，才能形成完整的月度画像。如果独立出去，就变成了单纯的记账工具，失去了"综合复盘"的价值。

### 为什么结余自动算？
减少心算负担。收入和支出填完，结余马上出来，不需要用户自己按计算器。

### 为什么不做数据导出/导入？
当前优先度不高。数据都在 Supabase 云端，不会丢。如果需要迁移，可以通过 Supabase 后台直接导出 CSV/JSON。未来如果有用户量或换平台需求再加。

### 为什么是单文件架构？
对非技术用户友好——没有 build 步骤、没有依赖安装、不用学 npm。一个 HTML 文件，浏览器打开就能用，改完 git push 就上线。代价是文件比较长（约 5400 行），但用代码分块注释 + 文档索引来管理。
