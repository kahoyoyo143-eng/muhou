## 技术备忘

### 添加新 Supabase 表字段的方法
直接用 Supabase Dashboard 的 SQL Editor 执行：
```sql
ALTER TABLE 表名 ADD COLUMN IF NOT EXISTS 字段名 类型;
```

### 文件结构
- 所有代码在一个 `index.html` 文件里（约 5400 行）
- `<style>` 标签在 `<body>` 前
- 业务 JS 分组在文件头部
- 页面初始化和路由在文件尾部
- 弹窗容器 `<div id="okr-modal-container">` 统一管理所有弹窗

### JS 代码分段（截至2026-06-23版本）

| 行号范围 | 内容 |
|----------|------|
| 8-500 | OKR + 习惯 + 今日3件事 + 备忘录（核心业务逻辑） |
| 501-3600 | CSS 样式 + 响应式 |
| 3600+ | HTML 结构 + 页面切换 + 仪表盘 + 复盘 + 愿景版 + 设置 |

### Supabase 表结构

| 表名 | 用途 | 关键列 |
|------|------|--------|
| okrs | OKR 目标/任务 | title, type(objective/kr/task/subtask), parent_id |
| habits | 习惯定义 | title, emoji, rest_days |
| habit_logs | 习惯打卡记录 | habit_id, date |
| diaries | 日记 | date, mood, content |
| inspirations | 灵感库 | title, status, tags |
| reviews | 月度复盘 | period, highlights, low_points, learnings, next_focus, income, expense, saved_this_month, saved_total, finance_note |
| visions | 愿景版便签 | tab(n-year/1-year), x, y, content, image_url |

### 部署
- GitHub Pages，legacy 模式（Branch: main, 根目录 `/`）
- `git push` 后自动部署，约 30 秒生效
- 不需要 GitHub Actions（legacy 模式不需要 workflow 文件）
