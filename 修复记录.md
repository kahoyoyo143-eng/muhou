# 修复记录

## 按时间倒序

### 2026-06-22：今日3件事 — 可选范围扩展 & 显示目标名

**需求：**
- 「从 OKR 任务中选择」只能选任务（task），不能选 KR 本身。如果 KR 下面没有子任务，就什么也选不了
- 选了之后只显示「OKR」标签，不知道来自哪个目标

**修改内容（`pickTodayItems` + `confirmPickToday` + `renderTodayItems` + 仪表盘）：**

1. `pickTodayItems()` — 现在同时收集 KR 和任务：
   - KR 本身始终可选（📋 前缀）
   - KR 下面的未完成任务也可选（✏️ 前缀，缩进显示）
   - 弹窗按 🎯目标 → 📋KR → ✏️任务 层级分组
   - 每个条目标记 `isKR: true/false` 和 `_idx` 序号

2. `confirmPickToday()` — 区分 KR 和任务：
   - 选 KR → 存 `krRef: { objId, krIdx }`
   - 选任务 → 存 `taskRef: { objId, krIdx, taskIdx }`
   - 两种都存 `objTitle`（所属目标名）

3. `renderTodayItems()` + 仪表盘 — 显示来源：
   - 以前：显示「OKR」标签
   - 现在：显示「← 成为有个人风格的短片导演」具体目标名
   - 自定义事项仍显示「自定义」

**覆盖的旧 bug：** 之前 KR 没子任务时弹「没有未完成的 OKR 任务」，现在不会了。

### 2026-06-22（早期）：新建目标保存按钮不工作 + 日期选择器失效

**根因1：** Supabase 的 `okrs` 表缺少 `deadline`, `memo`, `period_q`, `period_m` 四个字段

**修复1：** 执行 SQL 添加字段：
```sql
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS deadline text;
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS memo text;
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS period_q text;
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS period_m text;
```

**根因2：** 日期输入框 `<input type="date">` 在 Windows 11 浏览器上完全没反应（光标进不去，日历弹不出）

**修复2：** 改成 `<input type="text">` + placeholder 提示格式 + JS 正则校验：
```javascript
// 保存时校验格式
if (deadline && !/^\d{4}-\d{2}-\d{2}$/.test(deadline)) {
  alert('日期格式不对，请用 YYYY-MM-DD 格式，比如 2026-06-30');
  return;
}
```

### 2026-06-22（早期）：今日3件事选了但没保存

**根因：** `pickTodayItems()` 收集了任务列表但没存到 `window._pickTasks`，而 `confirmPickToday()` 从 `window._pickTasks` 读取 → 始终为空数组

**修复：** 在 `pickTodayItems()` 弹窗前加了 `window._pickTasks = tasks;`
