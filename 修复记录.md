### 2026-06-25：日记自动记录修复 + 灵感卡片改版 + 日记布局调整

**Bug：习惯打卡和灵感新增没有自动记到日记**
- **根因1**：`syncHabitToDiary()` 和 `syncTaskToDiary()` 只有当天**已有日记**时才写入 auto_entries，如果那天还没创建日记，记录就丢了
- **根因2**：灵感库完全没有日记同步功能
- **修复**：
  1. `syncHabitToDiary()` — 当天无日记时自动 upsert 创建一条，再写入 auto_entry
  2. `syncTaskToDiary()` — 同上修复
  3. 新增 `syncInspirationToDiary()` — 新建灵感/快速捕捉时自动同步到日记（`💡 捕捉灵感：xxx`）

**灵感库卡片改版：图片式 → 大标题式**
- **问题**：用户主要收集链接，卡片顶部 160px 图片区（或首字母占位符）浪费空间
- **修改**：
  1. 去掉 `.insp-card-img` 图片区域
  2. 标题从 15px 放大到 **18px 加粗**，最多显示 3 行
  3. 来源链接提到标题正下方，带浅蓝底色，显示域名（如 `🔗 example.com/article`）
  4. 描述字号从 12px → 13px，最多 2 行 → 3 行
  5. 偶尔有图片时，底部显示小缩略图（最高 120px），不抢标题风头

**日记页布局调整**
- **自动记录从右边挪到左边**：日历下方独立展示，右边日记区更干净
- **比例从 `280px 固定` → `3fr 7fr`**（中间试过 4:6 偏大，3:7 最合适）
- **日历字体随宽度等比放大**：日期 15px、心情 Emoji 15px、星期 12px

### 2026-06-22（早期）：新建目标保存按钮不工作 + 日期选择器失效

**根因1：** Supabase 的 `okrs` 表缺少 `deadline`, `memo`, `period_q`, `period_m` 四个字段

**修复1：** 执行 SQL 添加字段：
```sql
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS deadline text;
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS memo text;
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS period_q text;
ALTER TABLE okrs ADD COLUMN IF NOT EXISTS period_m text;
```

**根因2：** 日期输入框 `<input type="date">` 在 Windows 11 浏览器上完全没反应（光标进不去，日历弹不出）

**修复2：** 改成 `<input type="text">` + placeholder 提示格式 + JS 正则校验：
```javascript
// 保存时校验格式
if (deadline && !/^\d{4}-\d{2}-\d{2}$/.test(deadline)) {
  alert('日期格式不对，请用 YYYY-MM-DD 格式，比如 2026-06-30');
  return;
}
```
