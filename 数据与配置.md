## 数据库表结构

### `okrs` — OKR 目标

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid (自动生成) | 主键 |
| title | text | 目标名称，例如「成为有个人风格的短片导演」 |
| period_q | text | 所属季度，如 `2026-Q2` |
| period_m | text | 所属月份（可选），如 `2026-06` |
| deadline | text | 截止日期，格式 `YYYY-MM-DD` |
| memo | text | 备忘录/备注 |
| archived | boolean | 是否归档（默认 false），手动 toggle，可恢复 |
| key_results | jsonb | KR 数组，结构见下方 |
| created_at | timestamptz | 创建时间 |

### `habits` — 习惯

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| title | text | 习惯名称 |
| emoji | text | 图标 emoji |
| rhythm | text | `daily` 或 `weekly` |
| target_count | int | 每周目标次数 |
| weekly_days | int[] | 每周哪几天（如 [1,3,5]） |
| rest_days | int[] | 休息日（0=周日） |
| checkins | jsonb | 打卡记录 `{"2026-06-15": true}` |
| archived | bool | 是否归档 |
| created_at | timestamptz | 创建时间 |

### `daily_items` — 今日事项 + 备忘录（持久保留）

| 字段 | 类型 | 说明 |
|------|------|------|
| date | text (主键) | 固定为 `_current`，持久保留不按天清空 |
| items | jsonb | 事项数组（含今日事项 + 备忘录），结构见下方 |

### `reviews` — 月度复盘

| 字段 | 类型 | 说明 |
|------|------|------|
| period | text (主键) | 月份，如 `2026-06` |
| highlights | text | 高光时刻 |
| low_points | text | 低谷 |
| learnings | text | 学到什么 |
| next_focus | text | 下月重点 |

### `diaries` — 日记

| 字段 | 类型 | 说明 |
|------|------|------|

| mood | int | 心情 1-5 |

### `visions` — 愿景版

| 字段 | 类型 | 说明 |

| id | uuid | 主键 |

## 关键数据结构（JSONB）

### key_results（OKR 下的 KR）

```json
[
  {
    "title": "KR1: 完成3部短片",
    "tasks": [
      {
        "title": "写剧本初稿",
        "done": false,
        "doneAt": null,
        "subtasks": [
          { "title": "确定主题", "done": false },
          { "title": "写大纲", "done": false }
        ]
      }
    ]
  }
]
```

### daily_items.items（今日事项 + 备忘录，混合存储）

```json
[
  {
    "title": "写剧本初稿",
    "done": false,
    "source": "okr",
    "objTitle": "成为有个人风格的短片导演",
    "taskRef": { "objId": "xxx", "krIdx": 0, "taskIdx": 0 }
  },
  {
    "title": "回复合作邮件",
    "done": true,
    "source": "custom"
  },
  {
    "title": "确定主题",
    "done": false,
    "source": "okr",
    "objTitle": "成为有个人风格的短片导演",
    "subtaskRef": { "objId": "xxx", "krIdx": 0, "taskIdx": 0, "subIdx": 0 }
  },
  {
    "title": "买咖啡豆",
    "done": true,
    "doneAt": "2026-06-23T10:30:00.000Z",
    "type": "memo"
  }
]
```

引用类型说明：
- `krRef` — 直接选了 KR
- `taskRef` — 选了任务
- `subtaskRef` — 选了子任务（新增）
- `type: "memo"` — 备忘录项（新增，用于区分和复盘统计）
- 无引用 + 无 type = 自定义事项

### habits.checkins（打卡记录）

```json
{
  "2026-06-15": true,
  "2026-06-16": true,
  "2026-06-18": true
}
```

## 重要全局变量（JS）

| 变量 | 说明 |
|------|------|
| `sb` | Supabase 客户端实例 |
| `_okrList` | 所有 OKR 数据（内存缓存） |
| `_habitList` | 所有习惯数据 |
| `_todayItems` | 今日3件事数组（持久保留，不按天清空） |
| `_memoItems` | 备忘录数组（持久保留，`type: 'memo'`） |
| `_editingOKRId` | 编辑中的 OKR ID，null=新建 |
| `_editingHabitId` | 编辑中的习惯 ID，null=新建 |
| `_habitCalMonth/_habitCalYear` | 习惯月历当前查看的年月 |
| `_okrPeriodQ/_okrPeriodM` | OKR 筛选的季度/月份（月份默认空=全部） |
| `window._pickTasks` | 从 pickTodayItems 传给 confirmPickToday 的任务列表 |
| `_swYear` | 100件小事当前查看的年份 |
| `_swCategories` | 100件小事分类数组（内存缓存） |
| `_swItems` | 100件小事事项数组（内存缓存） |
| `_swEditMode` | 100件小事是否处于编辑/排序模式 |

## 新建表后必须执行的 SQL（关闭 RLS）

项目使用 publishable key，所有表都需要关闭 RLS 才能正常读写：

```sql
ALTER TABLE small_win_categories DISABLE ROW LEVEL SECURITY;
ALTER TABLE small_wins DISABLE ROW LEVEL SECURITY;
ALTER TABLE reviews DISABLE ROW LEVEL SECURITY;
```

如果忘记执行，写入操作会静默失败（不报错到前端）。
